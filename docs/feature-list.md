
# 功能清单（MVP 阶段）— 宠物社区 Demo

本文档用于明确当前 Demo 的**开发范围**与优先级，避免开发过程中频繁返工。
功能分为：
- MVP（本阶段必须完成）
- V1（下一阶段补充）
- V2（后续扩展）

---

## 一、MVP 范围声明

### 本阶段必须完成（MVP）
- 首页信息流展示
- 顶部宠物信息卡
- 内容详情页
- 发布图文动态
- 广告内容插入与展示
- AI 宠物助手（基础问答）
- 基础数据库与初始化数据
- 基础用户身份（可用模拟用户）

### 本阶段暂不做
- 推荐算法
- 私信聊天
- 视频上传与转码
- 支付与订单系统
- 商家后台系统
- 审核与风控体系
- AI 多模态识别（拍照/视频诊断）
- AI Agent 工作流（自动下单/自动预约等）

---

  - 品种

  - 标签

V1：
- 编辑宠物资料

V2：
- 健康数据管理

---

### 3. 商业广告位（MVP）
- 信息流中插入广告卡
- 广告类型：洗浴 / 宠物粮 / 宠物旅游
- 广告标识（推广）
- 点击跳转服务页

V1：
- 广告轮播
- 权重排序

V2：
- 精准广告推荐

---

### 4. AI 宠物助手（MVP）
MVP：先做“文本问答 + 风险分级建议”，支持与首页/我的页面入口打通。

- AI 问答入口（按钮/入口位）
- 文本问答（单轮/多轮）
- 输出结构化回答（便于演示）
  - 可能原因（最多 3 条）
  - 风险等级（低/中/高）
  - 建议（居家处理 / 需要就医）
  - 追问问题（最多 3 个）
- 对话记录（最小化：最近 N 条）
- 免责声明（非医疗诊断，仅供参考）
- 失败兜底与限流（提示重试/稍后再试）

V1：
- 结合“我的宠物信息”进行个性化（年龄/品种/体重等）
- 常见问题快捷提问（疫苗/驱虫/拉稀/呕吐等）
- 处置建议卡片化展示（可点开“附近医院/服务”）

V2：
- RAG 宠物知识库（权威资料引用）
- 多模态（上传照片做初筛：皮肤/粪便等）
- 与服务闭环联动（推荐医院/预约洗浴/购买处方粮）

## 三、MVP 验收标准

当满足以下条件，即认为 MVP 完成：

1. 首页能显示真实数据库内容
2. 顶部能展示宠物信息
3. 信息流可滚动加载
4. 能发布新动态并在首页看到
5. 信息流中出现广告内容
6. AI 宠物助手可正常问答，并返回结构化结果
7. 新环境通过 docker compose 可启动数据库
8. Flyway 自动创建数据库结构

# MVP 说明

MVP（Minimum Viable Product）意为：**最小可行产品**。

含义：
只实现最核心功能，使产品**可以运行、可以演示、可以被用户理解**，
而不是一开始就做完整复杂系统。

MVP 的目标是：
- 快速验证产品方向
- 快速形成可演示成果
- 降低开发复杂度
- 为后续迭代打基础

在本项目中，MVP 的目标就是：
> 用户进入首页，可以看到宠物信息、社区内容，并能发布内容、看到广告，并使用 AI 宠物助手完成基础问答。

当这些功能跑通，就算 MVP 成功。
